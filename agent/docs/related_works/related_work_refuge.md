# 関連研究調査メモ: ReFuGe (WWW '26)

## 論文情報
- **タイトル:** ReFuGe: Feature Generation for Prediction Tasks on Relational Databases with LLM Agents
- **著者:** Kyungho Kim, et al. (KAIST)
- **出典:** Proceedings of the ACM Web Conference 2026 (WWW '26)
- **リンク:** [arXiv:2601.17735](https://arxiv.org/abs/2601.17735) / [GitHub](https://github.com/K-Kyungho/REFUGE)

## 研究の概要
リレーショナルデータベース（RDB）上の予測タスク（例：顧客の解約予測）の精度を向上させるために、**LLMエージェントを使って有効な「特徴量（Feature）」を自動生成するフレームワーク「ReFuGe」**を提案した研究。
RDBは複数のテーブルに情報が分散しているため、予測に有効な特徴量を作るには複雑な結合や集約（例：「過去1ヶ月の平均購入額」など）が必要になる。ReFuGeは以下の3つのエージェントを用いてこれを自動化する。

1.  **Schema Selection Agent:** 予測タスクに関連するテーブルやカラムを選定する。
2.  **Feature Generation Agent:** 選定されたスキーマに基づき、有効そうな特徴量候補（SQLやPandasコード）を生成する。
3.  **Feature Filtering Agent:** 生成された特徴量を「推論（Reasoning）」と「検証（Validation）」の両面から評価し、有望なものだけを残す。

これらを反復ループ（Iterative Feedback Loop）で回すことで、徐々に精度を向上させる。

## 本研究（卒論）への導入・位置づけ

### 1. 引用すべき文脈
「LLMエージェントを用いたリレーショナルデータベース操作・活用」の応用事例として引用する。特に、**LLMが複雑なスキーマ構造を理解し、目的（この場合は予測精度向上）に合わせて適切な操作（特徴量生成）を行える能力**を示した研究として位置づける。

### 2. 決定的な違い（差別化ポイント）
「**Feature Engineering (分析・加工)**」か「**Data Synthesis (創造・生成)**」か、という目的の違い。

| 特徴 | ReFuGe (Kim et al., 2026) | 本研究 (卒論) |
| :--- | :--- | :--- |
| **目的** | 既存データからの**特徴量抽出**・予測精度向上 | 新規データの**創造**・UI/UX検証 |
| **入力** | 既存のRDB（実データ） | 架空のシナリオ・要件 |
| **出力** | 新しいカラム（特徴量）とそれを生成するロジック | 新しいレコード（行データ）群 |
| **LLMの役割** | データサイエンティスト（分析官） | クリエイター（脚本家・設計者） |

### 3. 論文内での記述案（第2章 関連研究）

> 「また、Kimら[Ref]のReFuGeは、リレーショナルデータベース上の予測タスクにおいて、LLMエージェントを用いて有効な特徴量を自動生成するフレームワークを提案している。彼らは、Schema SelectionやFeature Generationといった専門化されたエージェントを協調させることで、LLMが複雑なリレーション構造を理解し、高度なデータ操作が可能であることを示した。
> 本研究は、このような『LLMエージェントによるRDBの構造理解と操作』というアプローチを共有しつつ、それをデータの『分析』ではなく『創造（ゼロからの生成）』に応用するものである。ReFuGeが既存データから有用な情報を引き出すのに対し、本研究のエージェントは社会的文脈に基づいて、存在しないデータを整合性を保って生成することを目指している。」
