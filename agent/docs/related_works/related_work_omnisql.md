# 関連研究調査メモ: OmniSQL (VLDB '25)

## 論文情報
- **タイトル:** OmniSQL: Synthesizing High-quality Text-to-SQL Data at Scale
- **著者:** Haoyang Li, et al. (Renmin University of China, ByteDance)
- **出典:** VLDB 2025 (Proceedings of the VLDB Endowment)
- **リンク:** [arXiv:2503.02240](https://arxiv.org/abs/2503.02240) / [GitHub](https://github.com/RUCKBReasoning/OmniSQL)

## 研究の概要
Text-to-SQL（自然言語質問からSQLへの変換）モデルの学習用データを、LLMを使って大規模かつ自動的に生成するフレームワークを提案した研究。
Web上の表データ（Web Table）を起点に、架空のビジネスシナリオとDBスキーマを合成し、さらにSQLクエリと自然言語質問のペア、およびCoT（Chain-of-Thought）解説を生成する。これにより、250万件規模の学習データセット「SynSQL-2.5M」を構築した。

## 本研究（卒論）への導入・位置づけ

### 1. 引用すべき文脈
「LLMを用いたデータベース関連データの自動生成」という広義のカテゴリにおける**最新のSOTA（State-of-the-Art）研究**として引用する。

### 2. 決定的な違い（差別化ポイント）
OmniSQLは「**学習用データ（問題集）**」を作ることが目的であり、「**アプリ検証用データ（中身）**」を作ることではない。

| 特徴 | OmniSQL (Li et al., 2025) | 本研究 (卒論) |
| :--- | :--- | :--- |
| **生成の目的** | Text-to-SQLモデルのファインチューニング用学習データ | WebアプリのUI/UX検証・負荷試験用テストデータ |
| **生成物の主体** | DBスキーマ + 質問文 + 正解SQL | ユーザー、作品、プロジェクト、ログ等のレコード群 |
| **データの量** | 1DBあたり数件のサンプル行のみ | 1DBあたり数百〜数千件の密なレコード |
| **重視する整合性** | SQL構文の正しさ、質問とクエリの対応 | ユーザー行動の文脈、時系列、社会的関係性 |

### 3. 論文内での記述案（第2章 関連研究）

> 「データベース関連のデータ生成における近年の進展として、Liら[Ref]によるOmniSQLが挙げられる。彼らはWeb上の表データから多様なデータベーススキーマとSQLクエリを自動合成するフレームワークを提案し、Text-to-SQLモデルの学習リソース不足を解消した。
> しかし、彼らの手法はあくまで『モデル学習用のスキーマとクエリ』を生成することに主眼を置いており、各テーブル内に格納されるレコード（行データ）は、スキーマ構造を理解するための最小限のサンプル（数件程度）に留まっている。
> これに対し本研究は、アプリケーションのUI表示や検索体験の検証を目的として、データベース内に格納される『大量のレコード群』を、社会的文脈や時系列整合性を保った状態で生成することに特化している点で異なる。」

## 補足
- OmniSQLのソースコードはGitHubで公開されており、DB生成部分のロジック（`data_synthesis`）は参考になるが、本研究の目的（大量レコード生成）にはそのまま転用できないことも確認済み。
